# NQ Train / Dev / Test Protocol

This project now separates NQ policy/scorer training from held-out Qwen evaluation.

## Source Data

- `data_official_aligned/nq_test.jsonl`: BEIR NQ query/document rows aligned with official NQ-open answers. Rows without official answers are marked non-gold and are not used for official scoring.
- `data_official_benchmark_full/nq_test.jsonl`: 2,255 official-gold NQ rows selected from `data_official_aligned`. Fixed attacks are generated for five attack types under `data_official_benchmark_full/attacks/`.
- `data_official_benchmark_500/nq_test.jsonl`: the existing held-out NQ-500 evaluation set used by the current Qwen main table. This is preserved as the final test set.

## Split Construction

The split is generated by:

```bash
python scripts/prepare_nq_train_dev_test_split.py \
  --source data_official_benchmark_full \
  --test-source data_official_benchmark_500 \
  --output data_official_nq_split \
  --train-size 1500 \
  --seed 42
```

Protocol:

1. Keep the existing `data_official_benchmark_500/nq_test.jsonl` as held-out test.
2. Remove those held-out query/sample ids from `data_official_benchmark_full`.
3. Deterministically shuffle the remaining official-gold rows by query id hash.
4. Write 1,500 train rows and 255 validation rows.
5. Subset fixed attack files by split and attack type.

## Current Files

- Train: `data_official_nq_split/nq_train.jsonl` — 1,500 rows.
- Validation: `data_official_nq_split/nq_validation.jsonl` — 255 rows.
- Test: `data_official_nq_split/nq_test.jsonl` — 500 rows, identical protocol to the previous NQ-500 held-out benchmark.
- Split-specific attacks: `data_official_nq_split/attacks/<split>/nq_<attack_type>.jsonl`.
- Compatibility combined attacks: `data_official_nq_split/attacks/nq_<attack_type>.jsonl`.
- Manifest: `data_official_nq_split/nq_split_manifest.json`.

## Training Method

- Learned doc scorer training now uses `data_official_nq_split`, dataset `nq`, split `train`.
- PPO document policy training configs now use `data_official_nq_split`, split `train`.
- The Qwen-in-loop PPO config is `configs/main/nq_doc_policy_train_qwen_reward.yaml`.
- `NQDocumentMaskEnv` now prefers split-specific attacks, e.g. `attacks/train/nq_poisonedrag.jsonl`, before falling back to combined attack files.

## Evaluation Method

- Development/tuning should use `data_official_nq_split`, split `validation`.
- Final clean/attack Qwen reporting should keep using the held-out NQ-500 configs under `configs/main/official_benchmark_500_*.yaml` unless a new larger held-out protocol is explicitly declared.
- ACC/F1 are computed only on samples with official answers. ASR is computed against fixed attack `target_answer`.

## Important Caveat

Older checkpoints trained before this split may still carry benchmark-family leakage. For clean paper results, retrain both the learned doc scorer and PPO policy from this split before reporting final numbers.
